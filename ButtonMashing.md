# Button-Pushers/Mashers

## Summary

This page is about the FE-CTF FPGA challenges *button-pusher* and *button-masher*.

Use ICESTORM tools to decompile bitstream, simulate for a long time, identify subcircuits, and most importantly push/mash that button.

For `buttom-mashers` simulation `CXXRTL` was used for it to be speedy enough.

## Button-Pushers

To decompile the bitstream use:

```
iceunpack top.bin top.asc
icebox_vlog top.asc > top.v
```

This results in a Verilog file that has 2 inputs and 2 outputs. Looking at the outputs, it's clear that one of them is the inverse of the other, one can be ignored.

### Attempts at solving

1. *Try to use formal verification to get the output to flip.*

   This didn't bear fruit as it would later turn out that the debounce circuit has a very long settling time.

2. *Try and see if it needs glitching.*

   This didn't really show anything interesting.

### Final attempt: Just apply some random slow stimuli.

This worked and resulted in bitflips. Successive changes in the switch resulted in more bitflips on the output and looking at it it immediately produces a 0x7D, with subsequent bytes being the flag. **Success!**

```
#!/bin/bash

iverilog tb.v top.v
./a.out
```

## Button mashers

Same as the previous. Decompiling produced a slightly larger design, but the same interface as hinted by the schematic.

Applying the same slow or slower stimuli produced a reliable bitstream, but the bits didn't make sense.

### Attempts at solving

1. *Try out different serial protocols.*

   UART, NRZ line coding, etc. Nothing seemed to make sense.

### Final attempt: Actually look at all the signals, identify patterns and start reversing the verilog code.

Immediately it appeared that the design contained 2 separate carry chains (special ICE40 primitives), these could fairly easily be identified as a 6 bit and a 19 bit counter.

The small counter was seemingly only used for initialization, like a reset generator.

The large counter was used in the debounce circuit and counted down from 300000, providing what was assumed to be the shortest debounce timespan.

After that, by inspecting a random time in the simulation it is possible to notice a very wide LFSR waveform. Untangling this and renaming all uses of the LFSR bits in the Verilog file it's possible to find that one usage is to XOR the LED output with a bit of the LFSR. Removing the XOR operation with the LFSR a stream of characters is again emitted.

For timing the edge detection logic in the bitstream (`LUT 14 1 3`) seemed to work pretty well.

This worked somewhat but it only got us the last 42 characters of the flag. At some point it lost synchronization, without us being able to figure out why.
But with the part of the flag in hand we were able to guess the first part of the flag.

`main.cpp`
```c++
#include <iostream>
#include <fstream>
#include <iostream>

#include <backends/cxxrtl/cxxrtl_vcd.h>

#include "top.cc"

// #define DEBUG

int ch = 0, dig = 0, cnt = 1000;
char buf[1024];

static void shiftout(bool b)
{
    ch |= (!b?1:0) << dig;
    dig++;

    if (dig >= 8) {
        dig = 0;
        buf[--cnt] = ch & 0xFF;
        printf("%02x\n%s\n", ch, &buf[cnt]);
        ch = 0;
    }
}

int main() {
    srand(10);

    cxxrtl_design::p_top2 top;
    #ifdef DEBUG
    cxxrtl::debug_items all_debug_items;

    // Load the debug items of the top down the whole design hierarchy
    top.debug_info(all_debug_items);
   
    cxxrtl::vcd_writer vcd;
    vcd.timescale(1, "ns");

    vcd.add_without_memories(all_debug_items);

    std::ofstream waves("test.vcd");
    #endif

    bool test = true;

    bool curr, last = false;

    top.step();
    #ifdef DEBUG
    vcd.sample(0);
    #endif

    long step = 0;
    long iter = 0;

    auto run_stuff = [&](int iterations) {
        for (int i=0; i<iterations; i++) {
            top.p_clk.set(false);
            top.step();
            #ifdef DEBUG
            vcd.sample(step++);
            #endif
            top.p_clk.set(true);
            top.step();
            #ifdef DEBUG
            vcd.sample(step++);
            #endif

            iter++;

            curr = top.p_ce.get<bool>();
            if (!curr && last) {
                shiftout(top.p_o0.get<bool>());
            }
            last = curr;
        }

        #ifdef DEBUG
        waves << vcd.buffer;
        vcd.buffer.clear();
        #endif
    };

    top.p_sw.set(true);
    run_stuff(200);

    top.p_sw.set(false);
    run_stuff(1500000);

    for (int i=0; i<1000; i++) {
        top.p_sw.set(true);

        run_stuff(2500000/2);
        run_stuff(2500000/2);
        
        top.p_sw.set(false);
        run_stuff(2500000);
    }
}
```
